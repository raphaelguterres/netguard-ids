Agent + Server

A practical endpoint-to-SOC flow.

The agent collects host, process, network, and security telemetry, then sends normalized events to the central API using authenticated requests. The server handles ingestion, detection, risk, and incident context.

  • Host identity, heartbeat, telemetry, and local retry behavior.
  • API-key based ingestion and tenant-aware event processing.
  • Detection, correlation, risk scoring, and incident lifecycle foundations.
SOC Dashboard

Designed for analyst decisions.

The SOC workspace separates overview, inbox, incidents, host triage, agents, tenants, rules, and settings so demonstrations feel closer to a real product than a single crowded dashboard.

  • Overview with tenants, active hosts, open incidents, severity, and 24h timeline.
  • Operator Inbox ordered by risk with a clear triage call to action.
  • Host Triage view centered on posture, next action, evidence, MITRE context, and agent status.
